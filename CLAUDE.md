# XADO Кропивницький — інтернет-магазин автохімії та мастил

## Що це за проєкт

Інтернет-магазин продукції XADO (оливи, мастила, автохімія, автокосметика, технічні рідини, ревіталізанти). Зараз у процесі переходу зі статичного HTML на React/Next.js з подальшим підключенням Java-бекенду.

**Продакшн:** деплой на Netlify.
**Домен:** xado-krop.com.ua

## Поточний стан (as-is) — під час перехідного періоду

- Фронтенд: чистий HTML/CSS/JS, без білд-системи, без npm.
- "База даних": Google Sheets, опублікований як CSV (`pub?output=csv`), окремий `gid` на кожну категорію товарів.
- Дані завантажуються в браузері через PapaParse (`Papa.parse(sheetUrl, {download: true, header: true})`).
- Колонки в таблиці — українською: `ID`, `Назва`, `Категорія`, `Ціна`, `Обєм`, `Картинка`, `Опис`, `Переваги`, `Вимоги та допуски`, `Технічна інформація`, `Посилання`.
- Категорії товарів (=окремі HTML-каталоги і окремі gid у таблиці): `olyvy` (оливи), `mastyla` (мастила), `avtoXimia` (автохімія), `avtoKosmetika` (автокосметика), `technical_ridini` (технічні рідини), `revitalizant` (ревіталізанти).
- Картка товару рендериться динамічно через `products/{категорія}/product.html?id=XXX`.

## Куди рухаємось (to-be), поетапно

1. **Зараз:** очистка репозиторію від мертвого коду (старі дубльовані сторінки товарів, `*Old.html`, `.DS_Store`).
2. **Далі:** переписати фронтенд на **Next.js (App Router, TypeScript)**. Причина вибору Next.js замість чистого React SPA — потрібне SEO для ~170 товарних сторінок (SSR/SSG + `generateMetadata`).
3. **Пізніше:** свій бекенд на **Java** з базою на **PostgreSQL**, що замінить Google Sheets як джерело даних. Функціонал: кошик, оформлення замовлення, інтеграція з Новою Поштою та іншими перевізниками.
4. Шар отримання даних на фронтенді пишемо одразу через єдиний модуль (`lib/products.ts`), щоб заміна Google Sheets → Java REST API не вимагала переписування компонентів.

## Конвенції, яких дотримуємось при переписуванні

- Мова коду/змінних — англійська (навіть якщо назви колонок у Google Sheets українською — нормалізуємо їх у типізовані поля одразу на вході в `lib/`).
- Зображення товарів: перейменовувати в kebab-case латиницею без пробілів (поточні назви типу `"anyway 150.jpg"`, `"5w30 c3pro rozliv.jpg"` — проблемні для URL).
- Один компонент `ProductCard`, один шаблон сторінки категорії, один шаблон сторінки товару — замість окремого HTML-файлу на кожен товар/категорію.
- Не створювати нових захардкоджених product-сторінок — усе через `product/[id]/page.tsx` + дані з `lib/products.ts`.
- Header/Footer — спільні компоненти в layout, не копіювати в кожну сторінку.

## Відоме технічне сміття (на видалення при чистці)

- `products/*/*.html` (крім `product.html`) — ~170 застарілих статичних сторінок-дублікатів, замінені динамічним `product.html?id=`.
- `catalog/*/*Old.html` — старі версії каталогів.
- `.DS_Store` файли — сміття macOS, мають бути в `.gitignore`.
